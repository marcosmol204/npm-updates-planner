> **"Don't just run `npm update` and pray. Run a mission-critical audit first."**

This repository contains a high-fidelity LLM prompt designed to automate the **Research, Audit, and Planning** phases of a professional Node.js dependency migration. Instead of blind upgrades that break builds, this workflow generates a risk-aware execution roadmap.

---

## 🧠 The "Why"

In a production environment, upgrading dependencies is a **migration project**, not a command. 

I designed this prompt to replicate the exact manual process I follow as a Senior Developer:
1.  **Deep Scanning:** Manually searching the codebase for every `import` to prove a package is truly unused.
2.  **Ecosystem Mapping:** Identifying clusters (like Vite + Vitest + Plugins) that *must* move together to avoid peer-dependency hell.
3.  **Risk Assessment:** Cross-referencing CVEs and breaking changes before touching a single line of code.
4.  **Actionable Output:** Converting abstract "updates" into Jira-style tickets ready for a Sprint.

**This prompt encodes years of "manual" best practices into a single AI workflow.**

## 🚀 The Three-Phase Workflow

### Phase 1: The Deep Audit
The AI scans your `package.json` and build configs to detect unused packages, identify **End-of-Life (EOL)** software, and map out "Dependency Ecosystems."

### Phase 2: Actionable Ticketing
Findings are translated into structured **Jira-style tickets** categorized by priority:
* 🔴 **P1 Critical:** Active CVEs or broken builds.
* 🟠 **P2 High:** Framework migrations and performance gains.
* 🔵 **P3/P4:** DX improvements and "housekeeping."

### Phase 3: Flexible Implementation (Your Choice)
While the prompt defaults to generating a **standalone HTML dashboard** for stakeholder visualization, the output is modular. You can adapt the final stage to your specific stack:
* **Visual Reports:** Render a dark-mode HTML dashboard for a birds-eye view.
* **Direct Integration:** Use **MCPs (Model Context Protocol)** to automatically push generated tickets directly into **Jira**, **Linear**, or **GitHub Issues**.
* **Automation:** Pipe the JSON/Markdown output into custom CI/CD scripts.

## 🛠️ Usage Instructions

1.  **Copy** the contents of [`prompt.md`](./prompt.md).
2.  **Attach** your `package.json` (and `package-lock.json` if possible) to your LLM.
3.  **Provide** the prompt.
4.  **Review** the generated audit and ticket plan to populate your project management tool.
